Практична робота №7-8: Додавання JavaScript до мінісайту
Опис проєкту
Цей проєкт є фінальним етапом розробки адаптивного багатосторінкового сайту. На цьому етапі до статичного HTML/CSS інтерфейсу було додано клієнтську логіку на JavaScript: роботу з DOM, обробку подій, валідацію форм, використання LocalStorage та динамічне завантаження даних.
Мета роботи
Організація модульної структури JavaScript (ES6 Modules).
Реалізація інтерактивних UI-елементів (меню, акордеон, фільтрація).
Робота з браузерним сховищем (localStorage) для збереження стану (тема, чернетки).
Посилення UX через валідацію форм та лічильники символів.

Реалізований функціонал (10 обов'язкових завдань)
Модульність: Код розділений на окремі файли за логікою та підключений через type="module".
Активна навігація: Автоматичне підсвічування поточного пункту меню за URL.
Перемикач теми: Зміна світлої/темної теми з автоматичним збереженням вибору в localStorage.
Мобільне меню: Реалізовано функціонал "бургер-меню" з анімацією для смартфонів.
Акордеон: Інтерактивний блок питань-відповідей на головній сторінці.
Фільтрація: Динамічний фільтр карток послуг за категоріями без перезавантаження сторінки.
Модальні вікна: Універсальна логіка відкриття/закриття поп-апів.
Лічильник символів: Валідація довжини повідомлення в реальному часі в полі textarea.
Валідація форми: Перевірка введених даних перед відправкою та показ статус-повідомлень (успіх/помилка) в інтерфейсі.
Збереження чернетки: Текст повідомлення автоматично зберігається в localStorage при введенні та відновлюється після оновлення сторінки.

ПР9-10
У межах роботи було протестовано два підходи до створення HTML-елементів:

1. **Шаблонні рядки (Template Literals):**
   - *Використання:* Створення внутрішнього вмісту картки (заголовки, описи, бейджі).
   - *Переваги:* Код виглядає як звичайний HTML, його легко читати та редагувати. Дуже зручно вставляти змінні за допомогою `${variable}`.
   - *Недоліки:* Якщо вставляти дані через `innerHTML`, браузер щоразу перепаршує рядок, що повільніше при дуже великій кількості правок. Також це менш безпечно (XSS ризики).

2. **Метод `document.createElement()`:**
   - *Використання:* Створення зовнішньої оболонки картки та інтерактивних кнопок.
   - *Переваги:* Максимальна безпека та продуктивність. Найголовніше — ми отримуємо пряме посилання на об'єкт у пам'яті, тому можемо одразу повісити обробник подій (`addEventListener`) без пошуку елемента в DOM через `querySelector`.
   - *Недоліки:* Код стає дуже об'ємним («громіздким»), якщо створювати так великі структури (багато рядків `appendChild` та `textContent`).

**Висновок:** Для створення складних макетів зручніше використовувати **шаблонні рядки**, оскільки це швидше і наочніше. Проте для окремих функціональних елементів (кнопки, форми, інпути) краще використовувати **`createElement`**, щоб мати повний контроль над обробниками подій.
